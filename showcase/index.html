01

Описание и зачем

MailLab — учебная платформа для студентов, которые осваивают автоматизацию тестирования. Это веб-приложение из двух модулей: Почта (написать письмо, отправить, входящие, вложения) и Диск (папки, файлы, корзина).

Зачем своя платформа

Чтобы учиться писать автотесты, нужна мишень — сайт, который тестируешь. Настоящие Gmail, Яндекс Почта и прочие для этого не годятся: капча и защита от ботов, запрет на автотесты против прода, непредсказуемые данные. MailLab — наша платформа: всегда доступна, без капчи, с понятным состоянием. Можно регистрировать аккаунты, слать письма, наполнять диск — и проверять всё это тестами, ничего не сломав.

Для кого

  • Студенты курсов по QA-автоматизации — тренируются на реальных сценариях.
  • Преподаватель — даёт задания вида «напиши тест на регистрацию, отправку письма, корзину диска».
02

Подход и решения

Главная идея — приложение специально спроектировано как мишень для тестов. Это видно в трёх решениях.

Спецификация вперёд, разработка по TDD

Каждая фича сначала описана в отдельном spec-файле — эндпоинты, тела запросов, коды ответов, граничные случаи. Реализация идёт строго по TDD: сначала падающий тест (Vitest для API, Playwright для UI), потом код. В итоге у MailLab есть и эталонный набор тестов, с которым студент может сверять свои.

Предсказуемый API

REST под /api/, JSON, осмысленные статус-коды — никаких редиректов вместо ошибок. Контракты документированы, поэтому платформу одинаково удобно тестировать и через UI, и напрямую по API. 

POST /api/auth/register
{ "email": "ivan@maillab.local", "password": "..." }

201 → { id, email }
409 → { error: "email_taken" }
400 → { error, field }

Самодостаточность — никакой флакающей асинхронности

  • Реального SMTP/IMAP нет — почта смоделирована внутри. Письмо самому себе доставляется синхронно: тест отправил — оно сразу во «Входящих».
  • Корзина детерминирована: удалённый файл лежит 48 часов, затем фоновая чистка удаляет его окончательно — поведение легко проверить тестом.
  • Состояние сбрасывается командой — каждый прогон стартует с чистой базы.

04

Как пользоваться

Студенту нужен только браузер и аккаунт.

  1. Зарегистрироваться: email вида имя@maillab.local и пароль. Сразу появляется почтовый ящик и Диск с папками.
  2. Почта: написать письмо (кому, тема, текст, вложение), отправить, открыть во «Входящих», скачать вложение.
  3. Диск: создавать подпапки, перемещать файлы, удалять в Корзину и восстанавливать обратно.
  4. Писать автотесты на эти сценарии — через UI (Playwright, Selenium) или напрямую по API.
